## ğŸ”¬ Physics & Model

The dashboard focuses on the **GAAH model**, which exhibits a unique energy-dependent mobility edge separating localized and extended states.

### Hamiltonian
$$ H = -t \sum_{n} (|n\rangle\langle n+1| + \text{h.c.}) + \sum_{n} V_n |n\rangle\langle n| $$

### Quasiperiodic Potential
$$ V_n = 2\lambda \frac{\cos(2\pi b n + \phi)}{1 - \alpha \cos(2\pi b n + \phi)} $$

Here, b is set equal to $$ \alpha $$, a golden ratio.

### The Mobility Edge
The exact analytical boundary is given by:
$$ \alpha E_c = 2 \text{sgn}(\lambda) (|t| - |\lambda|) $$

## ğŸ“Š Dashboard Pages

1.  **Spectral Analysis**:
    *   Eigenvalue spectra ($E_n$).
    *   Level spacing statistics ($P(r)$) with Poisson vs. GOE indicators.
    *   Inverse Participation Ratio (IPR) distributions.

2.  **Dynamics**:
    *   **Mean Squared Displacement (MSD)**: $\sigma^2(t)$ to distinguish ballistic, diffusive, and localized transport.
    *   **Survival Probability**: $P(t)$ to measure return probability.
    *   **Participation Entropy**: $S_p(t)$ to visualize information spreading and multifractality.

3.  **Mobility Edge**:
    *   IPR vs. Energy scatter plots to pinpoint the edge.
    *   Eigenstate heatmaps ($|\psi_n|^2$) showing the coexistence of phases.
    *   Space-time evolution heatmaps ($|\psi(x,t)|^2$).

4.  **Fractal Dimension**:
    *   Estimates the correlation dimension $D_2 \approx -\ln(\langle \text{IPR} \rangle)/\ln(L)$.
    *   Identifies critical and multifractal regimes.

5.  **Global Trends**:
    *   Aggregated phase diagrams sweeping over potential strength $\lambda$.
    *   Maps order parameters like $\langle r \rangle$ and localized state fractions.

6.  **About**:
    *   Research motivation, detailed physics derivation, numerical methods, and references.
    *   **Call for Collaboration** for the Machine Learning phase of the project.

## ğŸ’» Numerical Implementation

The backend simulation (`gaahmodel_parallel_timeevol.py`) features:
-   **System Sizes**: Scales from $L=100$ up to $L=10,000$.
-   **Sparse Algebra**: Uses `scipy.sparse` (CSR format) for memory efficiency.
-   **Hybrid Diagonalization**: Switches between `eigsh` (Arnoldi) and `eigh` (dense) based on system size.
-   **Time Evolution**: Unconditionally stable **Crankâ€“Nicolson** integration.
-   **Parallelization**: `joblib`-based execution on multi-core CPUs.

> **Note**: If you identify any errors or potential issues in the Hamiltonian construction or the Crank-Nicolson time evolution code or physically inaccurate text or code implementation (e.g., results that seem physically inconsistent), please reach out to me. I am open to suggestions for modifications or updates to ensure the simulation's correctness.
